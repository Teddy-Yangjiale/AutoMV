## 做得好的部分

- 默认单句显示，歌词层级明确；
- 字体、颜色、遮罩和背景属于同一视觉方向；
- 句与句有连续入退场，不是静态 PPT 式切换；
- 当前六首音频整体偏低到中能，电影感宋体基线没有明显风格冲突。

## 主要问题

1. **缺少歌词语义证据。** 这是六首样本共同的最低维度。文件名和声学特征不能替代真实歌词。
2. **缺少段落级变化。** 固定的 `cinematic` 适合作为基线，但副歌和高潮没有相对提升，听觉在推进而画面仍保持同样呼吸速度。
3. **背景图尚未进入实测。** 当前只能评价压暗、循环和运动参数，无法确认中央区域的真实对比度与视觉噪声。
4. **BPM 存在半拍/倍拍歧义。** 分析器会折叠到更适合视觉运动的 70–160 BPM 区间，但低可信度结果必须人工听拍确认。

## 已采取的修正

- 增加八维自然度评分，缺少 LRC 时主动降低语义分；
- 增加音频情感导演和可解释的声学证据；
- 自动检测能量段落，为每段生成运动预设和强度；
- Studio 可导入 `director.json` 并实时预览段落切换；
- 最终 ASS 渲染端按每句所在段落选择动效，不再全曲固定一种运动；
- 大模型 prompt 同时接收声学画像和真实歌词，并被要求处理二者可能存在的情绪反差。
