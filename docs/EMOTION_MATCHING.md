# AutoMV 情感匹配与自然度管线

## 为什么此前的成片仍可能“不自然”

现有 Studio 已经能让背景、字体、颜色和句级动效保持统一，但它原先使用的是一套贯穿全曲的固定预设。这样做能避免视觉混乱，却无法表达主歌收敛、副歌抬升、间奏留白和尾奏回落。自然感的主要缺口不是再增加更多花哨特效，而是让同一视觉语法随音乐结构呼吸。

## 八个评价维度

| 维度 | 检查内容 | 当前自动化边界 |
|---|---|---|
| 情感方向匹配 | 整体视觉世界是否符合音乐的声学性格 | 可由音频初判，必须由歌词复核 |
| 字体气质 | 字形重量、衬线和手写感是否符合歌曲能量 | 可自动推荐，最终字体文件需人工确认 |
| 运动能量 | 位移、缩放、模糊和入退场速度是否符合节奏 | 可按段落自动控制 |
| 背景克制度 | 背景亮度、运动速度和循环周期是否让位于歌词 | 可检查配置，实际图片仍需审片 |
| 歌词可读性 | 字号、位置、密度、对齐和上下文数量 | 可自动检查 |
| 句间连续性 | 上一句离场和下一句入场是否属于同一运动轴 | 已由 ASS 交接保证 |
| 段落起伏 | 主歌、副歌、间奏和尾奏是否有层次 | 由音频能量段落自动化驱动 |
| 歌词语义证据 | 是否使用真实 LRC 校正音频推断 | 没有 LRC 时主动降分 |

## 情感导演如何工作

`emotion_director.py` 使用 FFmpeg 解码音频，再提取：

- BPM 与节拍可信度；
- 有效响度和动态范围；
- 频谱质心和过零率代表的音色明暗；
- 起音活动与节奏性；
- 约 10 秒以上尺度的能量变化点。

这些指标只形成“声学性格”，例如私密低能、温暖中能、清澈中能或明亮高能。它不会把明亮音色直接解释成快乐，也不会在没有歌词时声称理解歌曲故事。

分析结果包含：

```text
音频 → 声学特征 → 情感基线 → 统一视觉方向
                       ↓
                 段落能量曲线
                       ↓
       minimal / cinematic / float / punch / neon
```

整首歌仍只保留一个背景世界和一套字体家族。段落间只改变字幕运动、运动强度、强调色比例和背景速度，因此既有起伏，也不会像模板拼接。

## 标准工作流

先生成音频导演配置：

```powershell
python emotion_director.py "D:\Downloads\songs\月光(1).wav" `
  --offset 0 --output-dir benchmark\emotion
```

在 Studio 中导入生成的 `*.director.json`，视觉方向和段落自动化会一起应用。拿到 LRC 后，把声学画像交给大模型，让歌词语义修正背景意象：

```powershell
python mv_plan.py lyrics.lrc `
  --audio-profile benchmark\emotion\01_song.director.json `
  --output-dir plan\song
```

Studio 导出的 `automv-project.json` 会内嵌 `sectionAutomation`。最终渲染时，`render_project.py` 将每句歌词映射到所在段落，并由 libass 使用对应的动效和强度。

## 验收门槛

正式渲染全曲之前，必须先取一个安静主歌和一个高能副歌，制作 20–30 秒对照片段：

1. 关闭段落自动化导出 A 版；
2. 开启情感导演导出 B 版；
3. 检查歌词是否仍是视觉主体；
4. 检查副歌是否更有能量、主歌是否更克制；
5. 如果 B 版只是“更花”，而不是“更贴歌”，降低运动强度而不是增加特效。
